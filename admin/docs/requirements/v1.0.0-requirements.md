## 1. 项目概述

### 1.1 项目背景
基于现有 wujie 微前端框架，构建一套完整的后台管理系统，支持用户管理、角色权限、数据统计等核心功能。

### 1.2 技术栈
| 层级 | 技术选型 |
|------|---------|
| 前端 | React 19 + Vite + Tailwind CSS + shadcn/ui |
| 后端 | Node.js + Express.js |
| 数据库 | SQLite（better-sqlite3，本地文件存储，免费无需联网） |
| 认证 | JWT（jsonwebtoken） |
| 测试 | Vitest（单元/接口）+ Playwright（E2E） |

## 2. 功能需求

### 2.1 认证模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 登录 | 用户名+密码登录，返回 JWT Token | P0 |
| 登出 | 清除 Token，跳转登录页 | P0 |
| Token 刷新 | 自动续期，过期后重新登录 | P1 |
| 记住登录 | 可选"记住我"，延长 Token 有效期 | P2 |

### 2.2 用户管理模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户列表 | 分页展示用户，支持搜索/筛选 | P0 |
| 新增用户 | 填写用户名、邮箱、角色、状态 | P0 |
| 编辑用户 | 修改用户信息 | P0 |
| 删除用户 | 软删除（is_deleted 标记） | P0 |
| 重置密码 | 管理员为用户重置密码 | P1 |
| 用户状态 | 启用/禁用用户账号 | P1 |

### 2.3 角色权限模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 角色列表 | 展示所有角色 | P0 |
| 新增角色 | 创建角色并分配权限 | P0 |
| 编辑角色 | 修改角色权限 | P0 |
| 删除角色 | 删除角色（不允许删除有用户的角色） | P0 |

**内置角色：**
- `superadmin`：超级管理员，所有权限
- `admin`：管理员，用户管理权限
- `viewer`：查看者，只读权限

### 2.4 仪表盘模块

|------|------|--------|
| 统计卡片 | 用户总数、在线人数、今日登录次数 | P0 |

## 3. 非功能需求

### 3.1 性能
- API 响应时间 < 200ms（本地 SQLite）
- 前端首屏加载 < 2s（Vite 构建优化）

### 3.2 安全
- 密码使用 bcrypt 加密存储
- JWT Token 有效期 24 小时
- API 路由鉴权中间件保护
- 防止 SQL 注入（使用参数化查询）

### 3.3 兼容性
- 浏览器：Chrome 最新版、Safari 最新版、Firefox 最新版
- 分辨率：1280×800 及以上

## 4. 数据库设计

### 4.1 users 表
```sql
CREATE TABLE users (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  username   TEXT NOT NULL UNIQUE,
  email      TEXT NOT NULL UNIQUE,
  password   TEXT NOT NULL,          -- bcrypt hash
  role_id    INTEGER REFERENCES roles(id),
  status     TEXT DEFAULT 'active',  -- active | disabled
  is_deleted INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now'))
);
```

### 4.2 roles 表
```sql
CREATE TABLE roles (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL UNIQUE,  -- superadmin | admin | viewer
  label       TEXT NOT NULL,         -- 超级管理员 | 管理员 | 查看者
  permissions TEXT DEFAULT '[]',     -- JSON 数组
  created_at  TEXT DEFAULT (datetime('now'))
);
```

### 4.3 login_logs 表
```sql
CREATE TABLE login_logs (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id    INTEGER REFERENCES users(id),
  username   TEXT,
  ip         TEXT,
  status     TEXT,  -- success | failed
  created_at TEXT DEFAULT (datetime('now'))
);
```
