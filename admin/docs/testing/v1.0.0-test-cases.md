## 模块一：认证（Auth）

### TC-AUTH-001 正常登录
- **前提**: 系统运行，数据库已初始化
- **步骤**: POST /api/auth/login，body: `{username: "admin", password: "Admin@123456"}`
- **预期**: HTTP 200，返回 token 和用户信息
- **自动化**: ✅ backend/tests/auth.test.js

### TC-AUTH-002 错误密码登录
- **步骤**: 使用正确用户名 + 错误密码登录
- **预期**: HTTP 401，message: "用户名或密码错误"
- **自动化**: ✅

### TC-AUTH-003 禁用账号登录
- **步骤**: 使用 test_user2（disabled）登录
- **预期**: HTTP 403，message: "账号已被禁用"
- **自动化**: ✅

### TC-AUTH-004 无 Token 访问受保护接口
- **步骤**: 不带 Authorization header 访问 /api/users
- **预期**: HTTP 401
- **自动化**: ✅

### TC-AUTH-005 Token 过期
- **步骤**: 使用过期 Token 访问接口
- **预期**: HTTP 401，message: "Token 无效或已过期"
- **自动化**: ✅

## 模块二：用户管理（Users）

### TC-USER-001 获取用户列表
- **前提**: 已登录 admin，获得 Token
- **步骤**: GET /api/users
- **预期**: HTTP 200，data.list 为数组，data.total 为数字
- **自动化**: ✅

### TC-USER-002 分页查询
- **步骤**: GET /api/users?page=1&pageSize=2
- **预期**: list.length <= 2
- **自动化**: ✅

### TC-USER-003 关键词搜索
- **步骤**: GET /api/users?keyword=admin
- **预期**: 返回包含 admin 的用户
- **自动化**: ✅

### TC-USER-004 创建用户
- **步骤**: POST /api/users，body: `{username, email, password, roleId, status}`
- **预期**: HTTP 201，data.id 存在
- **自动化**: ✅

### TC-USER-005 用户名重复创建
- **步骤**: 使用已存在的用户名创建
- **预期**: HTTP 409
- **自动化**: ✅

### TC-USER-006 更新用户
- **步骤**: PUT /api/users/:id，修改 status 为 disabled
- **预期**: HTTP 200，用户状态已更新
- **自动化**: ✅

### TC-USER-007 删除用户（软删除）
- **步骤**: DELETE /api/users/:id
- **预期**: HTTP 200，用户从列表消失但数据库记录保留（is_deleted=1）
- **自动化**: ✅

### TC-USER-008 删除自己
- **步骤**: 以 admin 身份删除自己的账号
- **预期**: HTTP 400，message: "不能删除自己"
- **自动化**: ✅

### TC-USER-009 重置密码
- **步骤**: PUT /api/users/:id/reset-password，body: `{newPassword: "NewPass@123"}`
- **预期**: HTTP 200；使用新密码能登录
- **自动化**: ✅

### TC-USER-010 viewer 无权创建用户
- **步骤**: 以 viewer Token 发送 POST /api/users
- **预期**: HTTP 403
- **自动化**: ✅

## 模块三：角色管理（Roles）

### TC-ROLE-001 获取角色列表

- **预期**: 包含 superadmin, admin, viewer 三个内置角色
- **自动化**: ✅

- **预期**: HTTP 201
- **自动化**: ✅

### TC-ROLE-003 admin 无权创建角色
- **步骤**: 以 admin 身份 POST /api/roles
- **预期**: HTTP 403
- **自动化**: ✅

- **步骤**: 尝试删除 admin 角色（有用户使用）

## 模块四：E2E 用户流程

### TC-E2E-001 完整登录流程
- **工具**: Playwright
- **步骤**:
  1. 打开 http://localhost:5175/login
  2. 输入 admin/Admin@123456
  3. 点击登录
- **预期**: 跳转到 /dashboard，显示统计卡片
- **自动化**: ✅ frontend/tests/e2e/login.spec.js

- **步骤**: 输入错误密码并提交
- **预期**: 页面显示错误提示文字
- **自动化**: ✅

- **步骤**: 登录后点击「用户管理」
- **预期**: 显示用户列表表格
- **自动化**: ✅

### TC-E2E-004 退出登录
- **步骤**: 点击「退出登录」
- **预期**: 跳转到登录页
- **自动化**: ✅
